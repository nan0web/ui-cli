# Seed: Інтерактивна Допомога (OLMUI Interactive Content)

## 🎯 Мета

Перетворити статичну довідку на активний інтерфейс управління. Допомога стає повноцінним станом (State) всередині генератора моделі, де форма генерується динамічно.

## 🛠 Архітектурне бачення (Form Proxy & Context)

У межах нашої **Data Architecture**, ми хочемо, щоб опис інтерфейсу у YAML залишався чистим, а логіка «врізки» форми була автоматичною.

### 1. Опис у YAML (data/index.nan0)

Використовуємо семантичні прапорці та посилання на контекст:

```yaml
# Приклад опису сторінки допомоги у YAML
$content:
  - h1: 'nan0ai index'
  - p: 'Індексація робочого простору.'

  # Один вузол-проксі може приймати масив моделей.
  # $this - це автоматичне посилання на поточний екземпляр моделі, що виконується.
  # aiConfig - посилання на модель, яка завантажується через db.fetch('~/config')
  - form: [ $this, aiConfig ]
    summary: true # Показувати Live Preview рядок (команду)
    
  - p: "Натисніть кнопку нижче для запуску:"

  - button: Виконати
    $yield: 'execute'
```

### 2. Як виконується $yield (Алгоритм)

Коли користувач натискає кнопку з `$yield: 'execute'`, адаптер повертає цей результат у генератор моделі. Ось абстрактний код обробки:

```javascript
// Абстрактний цикл обробки інтентів в IntentDispatcher / Model.run
async *run() {
  while (this.help) {
    // 1. Рендеримо контент і чекаємо на дію користувача
    const response = yield ask('help', { UI: this.generateHelpAST() })
    
    // 2. Отримуємо payload з інтенту ($yield)
    const action = response.value?.$yield || response.value
    
    // 3. Логіка обробки конкретного yield
    if (action === 'execute') {
      this.help = false // Вимикаємо режим допомоги
      return yield* this.startTask() // Запускаємо основний процес моделі
    }
    
    // 4. Якщо це поле моделі — переходимо до редагування
    // Система автоматично знає, якій моделі (this чи aiConfig) належить поле
    const targetModel = this.resolveTargetModel(action)
    if (targetModel) {
      const fieldVal = yield ask(action, targetModel.constructor[action])
      if (!fieldVal.cancelled) {
         targetModel[action] = fieldRes.value
      }
    }
  }
}
```

### 3. Звідки беруться моделі ($this та aiConfig)?

- **$this**: Це зарезервоване слово в OLMUI, яке завжди вказує на поточну модель, що запустила `run()`.
- **aiConfig**: Модель завантажується автоматично через архітектуру **DB-FS**. Якщо вона вже була збережена в `~/config.yaml`, її поточний стан (наприклад, токен) буде підтягнутий при ініціалізації. Якщо ні — використаються `default` значення зі схеми.
