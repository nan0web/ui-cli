## 🌐 Vision: One Logic, Many UI

- **CLI/Web/React Parity**: Кожен компонент має працювати ідентично у кожному інтерфейсі.

## ✅ Form Validation (v1.3.0)

### Огляд

`Form` клас підтримує автоматичну валідацію введених даних на основі схеми моделі. При невалідному введенні користувач отримує повідомлення про помилку та можливість повторити ввід.

### Підтримувані валідатори

Form підтримує **два типи валідаторів** у схемі моделі:

- `validator` (пріоритет) - функція валідації з логікою перевірки
- `validate` (legacy) - альтернативна назва для сумісності

### Приклад використання

```javascript
export class Credit {
	static amount = {
		help: 'credit.amount.help',
		type: 'number',
		defaultValue: 10000,
		min: 1000,
		max: 200000,
		validator: (v) => !isNaN(Number(v)) && Number(v) >= 1000 && Number(v) <= 200000,
	}
	amount = 10000
}
```

### Поведінка валідації

1. **Повторний запит**: При помилці валідації Form автоматично повторює запит до користувача
2. **Повідомлення**: Виводиться через `console.error` з перекладеним ключем `validate.error`
3. **Типи відповідей валідатора**:
   - `true` - валідація пройшла
   - `false` - помилка з автоматичним повідомленням
   - `string` - кастомне повідомлення про помилку
   - exception - перехоплюється, показується `error.message`

### 🎚️ Автоматичний Slider для числових полів

Якщо поле має `type: 'number'` і визначені `min` та `max`, **Form автоматично використовує slider замість текстового вводу**:

```javascript
static amount = {
	type: "number",
	min: 1000,
	max: 200000,
	step: 100,  // опційно, default: 1
	validator: (v) => v >= 1000 && v <= 200000
}
```

При такій конфігурації користувач отримає інтерактивний slider для вибору значення в діапазоні 1000-200000.

### Приклад тесту

```javascript
const inputSequence = [
	'apps',
	'credits',
	'Test Credit', // title
	'abc', // amount invalid (type)
	'100', // amount invalid (range)
	'5000', // amount valid ✅
	// ...
]
// Result: File contains amount=5000, invalid values were rejected
```

### Інтеграція з перекладами

Рекомендується додати ключ `validate.error` у файл перекладів:

```javascript
// i18n.js
{
	uk: {
		'validate.error': 'Помилка валідації'
	}
}
```

## 📡 Future Interface Implementation Plan

Для реалізації стратегії **One Logic — Many UI**, ми плануємо впроваджувати інтерфейси у такій послідовності:

1. **Chat Interface (Base for Voice)** 💬
   - Будь-який додаток — це діалог. Форми в чаті стають послідовністю запитань.
   - Спільний `FormAdapter` транслює поля моделі у повідомлення чату.
   - `Select` стає списком кнопок або цифровим меню.
   - **Статус**: Логіка `ui-cli` вже готова до цього на 90% (послідовний ввід).

2. **Voice Interface** 🎙️
   - Розширення Chat Interface. Голос — це аудіо-транспорт для тексту.
   - Використання TTS (Text-to-Speech) для "рендерингу" та STT (Speech-to-Text) для вводу.
   - **Особливість**: Орієнтація на короткі відповіді та підтвердження.

3. **React / Lit Components** ⚛️
   - Генерація UI на основі тих самих JSON-схем, що і в CLI.
   - Використання бібліотек стилів (наприклад, Tailwind) для відповідності UX.
   - **Статус**: Спільна валідація та моделі вже можуть бути підключені.

4. **Static HTML / Forms** 📑
   - Традиційні веб-форми як fallback.

## 🧩 Future Refinements (CLI)

### 🎚️ Slider Improvements

- **Interactive Fix**: Потрібно розібратись з вильотом `SliderPrompt` в реальному TTY (конфлікт з `prompts`).
- **Smart Scaling**: Автоматичний вибір кроку (`step`) залежно від діапазону (0-100 -> 1, 0-1M -> 1000).
- **Keyboard Shortcuts**: Підтримка `+`/`-` для інтуїтивного контролю.
- **Multipliers**: Модифікатори (Shift/Ctrl) для прискореного руху (якщо дозволяє бібліотека).

### 🧪 Quality Assurance

- **Snapshot Coverage**: Додати снепшоти для сценаріїв відміни (Escape) та помилок вводу для кожного демо.
- **Escape Consistency**: Гарантувати, що `Esc` завжди повертає до меню або коректно завершує дію.

## 🔗 NanoWeb App Integration

- **CLI URL Access**: Перегляд сайтів як директорій `cd https://example.com`.
- **JSONL Sync**: Автоматичне завантаження індексованих даних сайту.
- **AI Co-pilot**: Використання LLM для аналізу та систематизації власних даних.
