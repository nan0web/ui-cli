# Release v2.8.0: Інтерактивні В'ювери та Розширені Режими Форм

## Scope
- Реорганізувати внутрішні компоненти UI-CLI відповідно до принципів Data-Driven OLMUI.
- Впровадити `FormViewer` (Menu-based editor та Inline Forms) і `ContentViewer` (Markdown viewer).
- `FormViewer` має підтримувати три поведінки:
   1. **«Агресивний» режим (Sequential)** - послідовне опитування кожного поля (так як працює зараз при `ask()` в `ui/form.js`).
   2. **«Пасивний» режим (Menu / Map Editor)** - відображення всіх полів відразу у вигляді меню / списку, де користувач вільно вибирає потрібне поле за допомогою стандартної навігації (Tab / Arrow Keys), оновлює його і натискає збереження або ескейп.
   3. **«Найбільш пасивний» режим (Summary Line)** - згорнута форма. Відображається як 1 рядок з підсумками (через кому). При натисканні Enter або зміщенні фокусу — форма розкривається в один з інших режимів.
- Всі нові контролери спираються на стандартні UI-доміки і `nan0web/ui-cli`.

## Acceptance Criteria
- [ ] Є контрактні тести `FormViewer`, що перевіряють рендеринг усіх трьох режимів (`aggressive`, `passive-menu`, `summary`).
- [ ] Навігація в `FormViewer` реалізована за стандартом доступності (Tab / Shift-Tab та ⬆ / ⬇ / ⬅ / ➡ стрілки).
- [ ] Фокусна навігація в `ContentViewer` вміє підсвічувати активні елементи.
