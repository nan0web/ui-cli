# 📸 CLI Gallery (nan0gallery) 

У рамках екосистеми OLMUI (One Logic — Many UI) ми обов'язково повинні генерувати візуальну галерею `CLI_GALLERY.md`, яка містить скріншоти (текстові зліпки) CLI-інтерфейсу. Вона використовується для мануального QA та ідеальної візуалізації.

## 🏗️ Структура галереї 

Галерея має сувору ієрархію, що генерується безпосередньо в директорії `snapshots/cli/`. Це дозволяє зручно переглядати всі снапшоти через вбудований Markdown-рев'ювер VSCode, уникаючи засмічення кореня проєкту чи папки `docs/`.

Структура для CLI виглядає так:
`snapshots/cli/{locale}/{component_or_flow}/`

У **кожній** папці генерується файл `index.md`, який має обов'язкове посилання `⬅ Назад` на рівень вище.

1. **Кореневий індекс `snapshots/cli/index.md`**:
   Знаходиться у корні CLI-галереї. 
   - Посилання на локалізовані галереї: `[uk](./uk/index.md)`, `[en](./en/index.md)`.
   - В самому кінці може йти посилання на "Головний екран" (Welcome / Root Router).

2. **Мовний індекс `snapshots/cli/{locale}/index.md`**:
   - Навігація вгору: `[⬅ Назад](../index.md)`.
   - Списки компонентів або User Stories. Посилання ведуть у відповідні папки.

3. **Сторінка компонента/флоу `snapshots/cli/{locale}/{component_or_flow}/index.md`**:
   - Навігація вгору: `[⬅ Назад](../index.md)`.
   - Містить усі текстові зліпки (`.txt` файли) або вбудовані ANSI-блоки коду, які генеруються після проходу тесту для конкретного сценарію.

## ⚙️ Як сформувати (Команди)

Як правило, всі OLMUI-пакети вже налаштовані на автоматичну генерацію через `packages/ui-cli/bin/nan0gallery.js`.

- 🚀 **Генерація Snapshots:**
  Для створення самих зліпків інтерфейсу використовується `npm run test:play`. Це запускає `cli.test.js`, який проходиться по всім User Stories та зберігає текстові логі в `snapshots/cli`.
  
- 🛠 **Складання Галереї:**
  Готова галерея генерується командою `npm run test:gallery` (або `npx nan0gallery --dir=snapshots/cli`). Markdown-індекси створюються автоматично.

- ✨ **Повний цикл:**
  Зазвичай `npm run test:all` містить і `test:play` і `test:gallery`.
  Якщо потрібно оновити існуючі снепшоти Playwright/CLI, можна використовувати `npm run test:play -- -u`.

## 🛡️ Правила верифікації
- **Zero Raw Keys:** У фінальній галереї не має бути сирих i18n-ключів (наприклад, `**cards.details.description**`), якщо це справжній UI. Всі ключі повинні успішно проходити процедуру перекладу (бути "bubbles up").
- **Ізоляція:** Зліпки не повинні залежати від локального оточення девелопера.
