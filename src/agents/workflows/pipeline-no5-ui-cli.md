# 🏭 `pipeline-no5-ui-cli` (CLI Adapter)

> **Призначення:** П'ятий крок конвеєра. Створення CLI-інтерфейсу на базі контрактів з Кроку 3.

## Інструкція для Агента

Коли Архітектор викликає цей workflow:

1. **Перевірка передумов (Zero-Hallucination):**
   - Модель (`src/domain/`) існує та 100% покрита тестами.
   - Контрактні тести (`src/test/*.test.js`) повністю зелені.

2. **Створення CLI Адаптера (Data-Driven UI):**
   - Створити `bin/run.js` (або `src/ui/cli/index.js`) як головну точку входу.
   - Використовувати стандартні генератори з `@nan0web/ui-cli` (напр. `renderForm`, `runGenerator`).
   - CLI **не повинен містити бізнес-логіки**. Тільки збирати вводи користувача в `[Intents]` та передавати Моделі.
   - Динамічні списки заповнювати виключно через metadata моделі.

3. **Стилізація:**
   - Кольори, відступи та розмітка беруться **тільки** з Design Token Manifest (`theme.js` або системних токенів). Жодного ANSI хардкоду.

4. **Тестування:**
   - Покрити CLI-адаптер snapshot-тестами (Golden Master / `olmui-scenario-test`).
   - Запустити `npm run test:all` — всі тести мають бути зеленими.

5. **Фінальна Перевірка:**
   - Запустити `node bin/inspect-arch.js` для перевірки, що архітектура не зламана.
   - Всі виводи CLI **мають використовувати** компоненти `@nan0web/ui-cli` (`Alert`, `Table`, `ProgressBar`), а не raw `console.log`. Див. `/ui-cli-standards`.

## Завершення:
Після створення CLI-інтерфейсу, запропонуй Архітектору зафіксувати версію та перейти до `/pipeline-no6-ui-chat`.
