# 🦅 UI-CLI Project & Architecture

**Призначення:** Пакет `@nan0web/ui-cli` забезпечує текстову репрезентацію "One Logic — Many UI" архітектури. Він перетворює Доменні Моделі на інтерактивні діалоги в терміналі.

---

## 🧭 Фаза 1: Філософія та Абстракція (The Seed)

1. **Місія та Філософія:**
   - **Взаємодія як Діалог:** Додаток — це цикл запитань і відповідей. CLI формалізує цей діалог через атомарні примітиви.
   - **Мінімалістична Істина:** Якщо бізнес-логіку неможливо виразити в CLI, вона ще не достатньо формалізована.
   - **LLM-First:** CLI компоненти слугують базою для ШІ-агентів, які використовують ці примітиви для керування системою.

2. **Абстракція (Базовий Додаток):**
   - Пакет розширює базові концепції `@nan0web/ui` та адаптує їх під обмеження `stdio`.
   - Використовує `InputAdapter.js` як уніфікований міст між логікою моделі та термінальними драйверами.

3. **Термінологія (Глосарій):**
   - **Prompt:** Інтерактивний вузол вводу (Input, Select, Confirm).
   - **View:** Статичний вузол виводу (Table, Alert, Badge).
   - **CGI (Command Graphic Interface):** Система рендерингу складних елементів у текстовому режимі.

---

## 📐 Фаза 2: Доменне Моделювання (Data-Driven Models)

1. **Створення Model.js:**
   - Моделі використовують JSDoc для опису типів, які `ui-cli` автоматично мапить на конкретні промпти.
   - Наприклад, `type: "Model"` автоматично відкриває рекурсивне меню вибору.

2. **Metadata Types:**
   - [x] `text` -> Input
   - [x] `boolean` -> Toggle/Confirm
   - [x] `enum` -> Select
   - [x] `collection` -> Multiselect/Tree
   - [x] `positional: true` -> Позиційний CLI аргумент (порядок = порядок оголошення `static` полів)

3. **Data Sources:**
   - Взаємодіє з YAML-сховищами через `@nan0web/db-fs`, забезпечуючи миттєвий відгук без накладних витрат на мережу.

---

## 🛠 Фаза 3: Верифікація Логіки (CLI-First)

1. **Інтеграція з `ui-cli`:**
   - Весь функціонал пакета доступний через `bin/cli.js`.
   - Команди будуються декларативно через `CommandParser.js`.

2. **Утиліти запуску (Core Runners):**
   - `runGenerator(model, adapter)` — з'єднує `async *run()` генератор моделі з `CLiInputAdapter`.
   - `runApp(EntryModel, adapter)` — цикл додатку для multi-command OLMUI систем.
   - `resolvePositionalArgs(Model, args, named)` — універсальний маппер CLI positional → model fields. Сканує `static` поля з `positional: true`, розподіляє за порядком оголошення. Іменовані опції мають пріоритет.

3. **Snapshot-тестування:**
   - Кожен компонент покритий 34 "Golden Master" снепшотами.
   - Снепшоти фіксують точний ASCII-вивід, включаючи ANSI-кольори та спецсимволи.

---

## 🪐 Фаза 4: Sovereign Workbench (The Master IDE)

1. **Підключення Models до Sandbox:**
   - `OlmuiInspector.js` — це інтерактивна пісочниця всередині терміналу.
   - Дозволяє змінювати `props` компонентів та бачити миттєвий результат рендерингу.

2. **Інтеграція Документації:**
   - `docs/en/` та `docs/uk/` містять глибоку документацію, яка інтегрована в довідку CLI (`--help`).

---

## 🎨 Фаза 5: Тематизація та Інтерфейси (Theming)

1. **Базовий UI та Брендування:**

   - Стилізація через ANSI-коди, що мапляться на системні палітри терміналів.

2. **Конфігурація:**
   - Підтримка Light/Dark режимів через адаптивні палітри Chalk/Colors.

1. **Snapshots & Verifications:**

   - Незалежний конвеєр `test:all`.

2. **Дистрибуція:**
   - Робочий npm-пакет з правильно експортованими типами (`@nan0web/ui-cli`).
   - `README.md.js` генерує еталонну документацію на основі живого коду.

## 💎 Фаза 7: Цінність та Економіка (Value & Economy)

1. **Цінність:** Розробник отримує готову "безкоштовну" адмінку/CLI до свого додатку просто правильно описавши доменну модель, без написання UI логіки.
2. **Ліцензія:** ISC — вільно і назавжди.

---

## 🌐 Фаза 8: Децентралізація та Суверенітет (Sovereignty)

1. **Автономність терміналу:** CLI компоненти не залежать від браузера, інтернету чи UI-бібліотек. Локальний комп'ютер є самодостатнім сервером рішень.
2. **Текстовий базис:** Будь-яка складна система керується текстом та конвеєрами POSIX, що дає суверенний контроль автору.

---

## 🌍 Фаза 9: Місія та Вплив (World Impact)

1. **Місія:** Довести, що навіть без графіки будь-який додаток може мати логічно досконалий інтерфейс, відшліфовуючи лише "Суть" (потоки даних).
2. **Вплив:** Інструмент стає невидимою мовою для спілкування Agent-to-Human (`prompts`) та Human-to-Agent (зрозумілі структури Models).
